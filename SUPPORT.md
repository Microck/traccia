# Support

Use the GitHub issue templates for bugs and feature requests.

Before opening an issue, include the smallest reproduction you can:

- Traccia version or commit.
- Operating system and Python version.
- The command you ran.
- Relevant config with secrets removed.
- Expected behavior and actual behavior.

Do not attach private archives, raw personal exports, credentials, or generated
private graph artifacts. If a reproduction needs source data, create a minimal
fixture that preserves the shape of the problem without exposing private
content.

Security issues should follow `SECURITY.md` instead of public issues.
